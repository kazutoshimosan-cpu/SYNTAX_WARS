# Netlify → GitHub 周りのメモ（Syntax Wars / 格ゲー）

自分用。専門用語はなるべく使わない書き方。

---

## いま何がどうなっているか

| 場所 | 役割 |
|------|------|
| **GitHub**（リポジトリ `SYNTAX_WARS`） | コードの保存・履歴。パソコンから **プッシュ** で更新できる。すでにこのフォルダは GitHub とつながっている。 |
| **Netlify** | インターネット上の「公開の窓口」。いまは主に **`syntax_wars.v2html.html`** を開く形でデプロイ。修正の主軸は **`syntax_wars_v4html.html`（V4）**。あとで V2 はやめる予定。 |

「GitHub に乗り換える」ときの本当の意味は、だいたい **「公開の窓口」を Netlify から、GitHub の Pages などに移す」** こと。コードそのものはもう GitHub にある。

---

## なぜ移したいか（自分向け）

- Netlify は **何度も載せ替える**と手間やミスが出やすいと感じている。
- GitHub に **プッシュしたら自動で公開も更新**、にすると、試行錯誤に強くなる（慣れれば）。

---

## 移すときにやることリスト（順不同でもOK）

1. **GitHub 側で「Web として出す」設定をする**  
   （GitHub Pages など。正確なクリック手順は、そのときの公式ヘルプか Cursor で聞く。）

2. **新しい公開 URL が決まったら**  
   - **Firebase** を使っているなら、Firebase の管理画面で **「この URL からもアクセスしていい」** と登録する（承認済みドメインなど）。やらないとログやオンラインが動かないことがある。

3. **古い Netlify の住所に来た人向け**  
   - Netlify に載せる画面を **「引っ越しました。新しいリンクはこちら」** の短いページに差し替える。  
   - ブックマークの人が迷わない。

4. **新しい GitHub 側のゲーム画面（任意）**  
   - 小さく「以前は Netlify でした」と書いてもよい。

5. **SNSやフォームに貼っている URL**  
   - 新しい住所に差し替える。

---

## Cursor から GitHub に送る（プッシュ）

- 左の **ソース管理（枝分かれマーク）** からコミット → プッシュ（同期）でもよい。
- 下の **ターミナル** で `git` コマンドでもよい。ターミナルだけ触っている間は **AI のトークンは減らない**（ローカルの作業）。

認証で詰まったら（Permission denied など）、そのとき SSH か HTTPS の設定を一緒に見ればよい。

---

## いつやるかの目安

- **V4 を本番に載せ替えるタイミング**にまとめてやると、Netlify の整理とホストの整理が一度で済む。
- 急がなくてよい。**Netlify をすぐ消さず**、GitHub 側だけ先に試してから切替、でもよい。
