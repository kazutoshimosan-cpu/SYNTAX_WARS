## 5. カプセル／ライブラリ画像（「DEMO」表記入り）★要作成

Steam は**デモのカプセルに "demo" が分かる表記**を求める。本編カプセルに「DEMO」帯を足した版を用意する。

- 元 = `~/Desktop/SYNTAX_WARS素材/steam_capsules/`（本編の各サイズ）。
- そこへ **右上などに「DEMO」リボン**を重ねた `*_demo.*` を作る（本編の `capsule_build.sh` と同じ ImageMagick の軽い上書き＝**重い処理なし・メモリ安全**）。
- 必要スロット＝本編と同じ（Header 920×430／Small 462×174／Main 1232×706／Vertical 748×896／Library 600×900）。フォーマットは §本編パケット6と同じ（大物JPG・小型/ライブラリPNG）。
- **タグライン焼き込みは引き続き禁止**（本編で一度リジェクトされた地雷）。**"DEMO" の識別表記は別物＝これは required**。

> ▶これはエージェント（俺）が用意できる device-free・軽作業。指示くれたら `capsule_build.sh` を土台にデモ帯版を焼く。

## 6. デモのビルド（遊べる中身）を上げる ★ここが実は"初めての作業"

> ⚠️ **現状認識（2026-07-23）**：Steam の置き場（depot）に**遊べるビルドを上げたことはまだ一度もない——本編もデモも**。ストアの「近日登場」ページはビルド無しで出せたので後回しになっていた。つまり **depot アップロードは初通し**で、手本になる本編アップロードは存在しない。ただしデモと本編は**同じ道具・同じ手順**なので、**デモで一度通せば発売（本編）でもそのまま使える**＝デモが本番の予行演習になる。

**道具＝SteamPipe**（Steamworks SDK 同梱の content builder ＋ steamcmd）。流れ：

1. **デモビルドを CI で焼く**：`build-game.yml` 実行 → `SyntaxWars-windows-demo` / `-linux-demo` / `-macos-demo` を取得（もーさんの機械は無負荷）。中身は3体・対CPU/2P・締めの壁つきに自動で絞り済み（本編コード非破壊）。
2. **Steamworks SDK を落とす**（中に steamcmd と content builder が入っている）。
3. **アップロード設定（.vdf）を書く**：どの depot に・どのフォルダの中身を上げるか。→ **これはエージェント（俺）が雛形を用意できる**（デモ App ID と depot ID が分かり次第）。
4. **steamcmd でログイン → アップ**：ここは **Steam Guard（2段階認証）が要る＝もーさんの手**（アカウント＋スマホ）。`steamcmd +login … +run_app_build app_build.vdf` を一度通す。
5. 上がったビルドを Steamworks の Builds でデモの**既定ブランチ**に割り当てる。

> mac 版は公証済みzip（[[project_mac_notarization]]）を載せる話・急がない。まず Win/Linux で通すのが最短。
> ★将来は GitHub Actions で自動アップ（`steam-deploy` 系）も可能だが、Steam 認証情報を CI に預ける判断が要る＝発売後に検討。

## 8. 審査 → 公開 → 本編ページにボタンを点ける

1. デモ App のランディングページで**各チェックリストを埋める** → **「Mark as ready for review（審査に出す）」**。
2. Valve 審査（**3〜5営業日**）。※公開希望日の**7営業日前**には出す＝手直し余裕。
3. 審査通過後、デモを **「coming soon」or「そのまま遊べる公開」** で出す（Next Fest 用は**遊べる公開**）。
4. **★本編ページを"再公開（republish）"しないと「デモをダウンロード」ボタンが出ない**：
   - 本編 App `4939710` → **Edit Store Page → Special Settings → Associated Demos** で表示を ON。

   - 設定したら**本編ストアページを再公開**（これでボタンが点灯）。
