## 1. ゲーム概要

| 項目 | 内容 |
|------|------|
| ジャンル | SF格闘ゲーム（ターン制コマンド選択） |
| プラットフォーム | ブラウザ（HTML / CSS / JavaScript） |
| プレイ人数 | 1人（vs CPU） / 2人（同端末 or ネット対戦：Firebase予定） |
| CPU難易度 | EASY / NORMAL / HARD |
| 操作方法 | ボタン選択→確定（READY）の2ステップ |
| 実装フェーズ | Phase 1〜5の段階実装（詳細は章9） |

### 基本ルール
- **行動選択：同時選択・速度順解決**。両プレイヤーは同じターンに行動を同時選択し、選択確定後に**SPEED順**（速い方から）に行動が解決される。
- **速度順解決の詳細：** SPEED差が10以上ある場合、先手の攻撃で後手のHPが0になったとき、後手の行動はキャンセルされる（相打ちにはならない）。SPEED差10未満の場合は同時解決（旧ルール）。
- **勝利条件：** 相手のHPを0にした方が勝利。
- **引き分け：** SPEED差10未満で同時解決のターンに両者HPが0になった場合は引き分け。リザルト画面に「DRAW」を表示し、リマッチ or タイトルへ戻るを選択できる。
- **試合形式：** 1本勝負。

## 2. 行動システム

### 2-1. ボタンレイアウト（4列）
※ スマホ対応のため各列は2×2グリッドで表示

| 列 | カテゴリ | 内容 |
|----|----------|------|
| 1列目 | 通常行動 | 弱・中・強・ガード（BLAZEのみ「起爆準備」）※技名はキャラごとに異なる |
| 2列目 | 属性付き攻撃 | 弱+速度特攻・中+ガード破壊・強+回復封じ・カウンター |
| 3列目 | 特殊行動 | 加速・スロー・回復・挑発・オーバークロック |
| 4列目 | 固有技 | キャラクター専用の必殺技（各1種） |

### 2-2. 通常行動（ゲージ不要）

技名はキャラごとに異なるが、ベースの数値は共通。キャラ補正がかかって最終ダメージが変わる。

| 行動 | ベースダメージ | 命中率 | ゲージ回収 | 追加効果 |
|------|---------------|--------|-----------|---------|
| 弱 | 8 | 100% | +15% | なし（確実なトドメ・序盤のゲージ溜め主軸） |
| 中 | 15 | 80% | +8% | なし（標準ダメージソース） |
| 強 | 22 | 70% | +5% | ヒット時：相手SPEED-10 / ジャストガードされると相手ゲージ+20% |

**全キャラ共通の回避ルール（v5新規）**
- 相手のSPEEDが自分より20以上高い場合、自分の「強攻撃」は確定で外れる。
- **SHADOW専用ボーナス（絶影）：** 速度差によって相手の強攻撃を回避した際、SHADOWのゲージが+20%される。

**キャラ別の技名（例）**

| キャラクター | 弱 | 中 | 強 |
|--------------|----|----|-----|
| NEXUS-7 🤖 | ナノブレード | EMP波動 | プラズマカノン |
| SHADOW 🥷 | クナイ投げ | 鎖鎌 | 影斬り |
| NOVA QUEEN 🔮 | エナジー弾 | 次元裂き | 重力場 |
| TITAN 🦾 | 鉄拳 | 地震砕き | ギガントプレス |
| LYRA-X 🧬 | 酸液噴射 | 変異爪 | 捕縛触手 |
| BLAZE 🔥 | ファイアボール | 火炎放射 | マグマストライク |

### 2-3. 属性付き攻撃（ゲージ25%消費 / SPEED差回避なし）

「必中」の定義（v5変更）：SPEED差による回避が不可能という意味。ガードによるダメージ半減・カウンターによる反射は有効。

| 行動 | ダメージ | 属性効果 | 対策できる相手の戦略 |
|------|----------|----------|----------------------|
| 弱 + 速度特攻 | 8 + SPEEDボーナス | 相手のSPEEDが高いほど追加ダメージ増加（機雷的役割） | 加速ゴリ押し |
| 中 + ガード破壊 | 15 | ガードを無視してダメージ | ガード多用 |
| 強 + 回復封じ | 22 | 次ターン回復・挑発を封じる（1ターン） | 回復待ち |
| カウンター | — | 相手の攻撃を受けた際に同等ダメージを返す。自分のSPEEDが低いほど自分も被ダメあり | 攻撃力の高いキャラ全般 |

**カウンターの詳細仕様**

- **発動条件：** カウンター選択ターンに相手が攻撃してきた場合に発動（全攻撃に反応）
- **返すダメージ：** 相手の攻撃ダメージと同等（等倍返し・ヒット率80%）。ただし**固有技に対しては返しダメージが半分（50%）**になる（v6変更）
- **自分が受けるダメージ：** 相手の攻撃ダメージ × SPEED差に応じた割合

| 相手SPEED − 自分SPEED | 自分が受けるダメージ |
|----------------------|-------------------|
| 0以下（自分が同速or速い） | 0%（完全シャットアウト） |
| 1〜19 | 0% |
| 20〜39 | 10% |
| 40〜69 | 20% |
| 70以上 | 30% |

**カウンター時の追加効果（デバフ）処理ルール（v5.1新規）**

カウンター成功時、攻撃に付随するデバフ（速度特攻のSLOW、回復封じなど）の適用は以下のルールに従う。

| 自分が受けるダメージ | デバフ適用 |
|---------------------|----------|
| 0（完全シャットアウト） | **無効化**（ダメージを0にしきった場合、攻撃はすべて弾かれる） |
| 1以上（かすりダメージあり） | **適用**（ダメージが貫通した場合、攻撃はヒットしたとみなす） |

コンセプト：SPEED差が大きい相手の速度特攻に対して完全カウンターができる状況では「肉を切らせず骨を断つ」ことができるが、SPEED差が小さいと貫通ダメージとともにデバフも受けてしまう。

**カウンターの連続使用ペナルティ（v5新規）**
- カウンターを連続ターンで使用した場合、ゲージコストが倍（25%→50%）になる。
- 連続使用カウントは攻撃または他の行動を1回行うとリセット。

※ LYRA-Xの回復封じは1ターン持続（v6変更：旧2ターン）
※ NOVA QUEENの属性付き攻撃は次ターンのゲージコストが半減（ゲージ60%以上時・v6変更：旧70%以上）

### 2-4. 特殊行動（ゲージ30%消費）

| 行動 | 効果 | 備考 |
|------|------|------|
| 加速 | 自分のSPEEDを+20〜30上昇（初期50のキャラが1回でFAST到達） | FAST状態になると速度特攻で追加ダメージを受けるリスクあり |
| スロー | 相手のSPEEDを大幅に低下 | 全キャラ共通1ターン持続（v6変更：LYRA-X特権廃止）。SLOW状態は相手ダメージ-20% |
| 回復 | HPを15〜25回復 | 回復封じ中は使用不可 |
| 挑発 | 次の自分の攻撃ダメージ+30% | 回復封じ中は使用不可 |
| オーバークロック | ゲージ100%消費で次ターン強制2回行動 | 使用後3ターンはゲージ増加量が50%に減少 |

### 2-5. 固有技（ゲージ50%消費 / SPEED差回避なし）

「必中」の定義（v5変更）：SPEED差による回避が不可能という意味。ガードによるダメージ半減・カウンターによる反射は有効。

| キャラクター | 固有技 | 効果 |
|--------------|--------|------|
| NEXUS-7 🤖 | 量子ビーム | **45ダメージ**・命中時ゲージ**+25%**・「量子強化」（次ターン属性攻撃コスト0%、1回限り）付与（v6.4：40→45） |
| SHADOW 🥷 | 影分身 | 次の相手の攻撃を1回**身代わり**で無効化（相手の攻撃は命中するが分身が受けるためダメージ・追加効果が0になる） |
| NOVA QUEEN 🔮 | 星屑砲 | **基礎40ダメ＋損失HP×0.5の追加ダメ**（v6.4：×0.55→×0.5）。相手ゲージ**-40%**（CROWD ON FIRE時-60%）・相手の属性攻撃を1ターン封じる |
| TITAN 🦾 | 超電磁砲 | 自分の残りHPに応じてダメージが変動（下記テーブル参照） |
| LYRA-X 🧬 | 細胞再生 | 与ダメージ30%をHP吸収 |
| BLAZE 🔥 | 爆裂拳 | **固定45ダメージ**（起爆準備後は×1.3=58）。確定自爆あり（与ダメ×0.2）。起爆準備で自爆回避（v6.1変更） |

**TITANの超電磁砲ダメージテーブル（v6.9調整）**

| 自分の残りHP | ダメージ |
|-------------|---------|
| 50以上 | 30 |
| 30〜49 | **45**（旧50） |
| 29以下 | **55**（旧75） |

コンセプト：相手は「TITANのHPを55以下に保ちたい」という意識が生まれ、TITAN戦の読み合いの軸となる。挑発込みで55×1.3＝71ダメも出るため瀕死時の脅威は健在。

**SHADOWの影分身について（v5変更理由）**
旧仕様「次の攻撃を完全回避」は固有技の「SPEED差回避なし」ルールと矛盾するため、「身代わり」方式に変更。相手の攻撃は命中判定を通るが、SHADOW本体へのダメージ・追加効果がすべて0になる。

### 2-6. ジャストガード
相手が「強攻撃」を選択したターンに自分が「ガード」を選択していた場合に発動。
- 被ダメージを0にする
- 自分のゲージを+20%する

※ BLAZEは「起爆準備」がガードの代わりのため、ジャストガードは発動しない
※ カウンター選択中はジャストガードと干渉しない（別の効果として独立）

### 3-2. パッシブ特性（全キャラ）

**NEXUS-7 🤖 ／ アドバンスド・ラーニング**
- 相手が属性攻撃をヒットさせると、NEXUS-7側に「学習済み」フラグをセット（v6.1バグ修正：旧実装はセット処理が欠落していた）
- 学習済み状態で同じ属性攻撃を使うと威力×1.8（+80%）。発動時にリセット（使うまで保持）（v6.4：旧×1.7）
- CROWD ON FIRE時は学習倍率が×2.0に強化
- NEXUS-7が属性攻撃を使用すると、学習ボーナスとは別に**×1.1の固有バフ**が追加（v6.1強化）
- **相手の固有技被ダメ-20%**（v6.4新規：相手がuniqueを使用した際のダメージ軽減）
- ※ 量子ビーム命中時の「量子強化」（コスト0%）とは別ステータス

---

**SHADOW 🥷 ／ 絶影（ぜつえい）**
- **全キャラ共通の速度差回避ルール（SPEED差20以上で強攻撃確定外れ）が発動した際、SHADOWのゲージが+20%される**（v6.4：旧+25%。v5.1変更：旧「SPEED差30で確定回避」→共通ルールに統合）
- **通常攻撃ダメージ実質-19%**（v6.8：dmgMod×0.9＋通常攻撃専用×0.9の二重補正。属性攻撃はdmgMod×0.9のみで-10%。速度特攻のボーナスダメージには適用しない）
- 2回行動時の2回目の攻撃は必ずクリティカル（1.3倍）
- **SPEED自然回復なし：** 毎ターン終了時のSPEED50への自然回復が発動しない。初期SPEED65を維持し続ける。
- コンセプト：速さを軸にした読み合い特化キャラ。「あえて加速して相手の強攻撃を誘いゲージを稼ぐ」戦略と、「速すぎると速度特攻の的になる」ジレンマが生まれる。

---

**NOVA QUEEN 🔮 ／ コズミック・共鳴（瀕死の逆転王）**
- ゲージが**60%以上**の時、属性付き攻撃のコストが25%→15%に半減（v6変更：旧70%以上）
- **序盤パッシブ（HP48以上）：被ダメージ-10%**（v6.4新規：序盤の生存力を底上げ）
- **瀕死バフ（HP40%以下・HP38以下）：（v6.4調整：旧HP33以下）**
  - 被ダメージ**-25%**
  - ゲージ増加量**+50%**（一般ルールのHP40%閾値とは独立した固有ルール）
- コンセプト：「あと1発で倒せる」状況が最も危険なキャラ。追い詰めた側は「確実に仕留める」かCROWD ON FIREを警戒するかの判断が必要。
- **CROWD ON FIRE時ボーナス：** 星屑砲のゲージ吸収が-40%→**-60%**に強化
- ※ 実装時注意：ゲージ60%の閾値・HP38の閾値・HP48の閾値を毎ターン判定し、HUDをリアルタイムで更新すること

---

**TITAN 🦾 ／ 不動の要塞**
- SLOW状態（SPEED 35以下）の間、被ダメージを常時**15%カット**（v6.9調整：旧25%）
- **SLOW状態の間、全キャラ共通の「与ダメージ-20%」ペナルティを完全に無効化し、さらに与ダメージ+10%の専用バフを得る（v6.9調整：旧+20%）**
  - コンセプト：「遅ければ遅いほど一撃が重くなる」重戦車。相手に「スローを使うと硬化＆火力UPを許すジレンマ」を強要する。
  - HUD表示：SLOW状態のTITANには「SLOW⚡+10%」タグを表示
- カウンターと相性が良い：低SPEEDでもダメージ覚悟で大ダメージを返せる
- **オーバーヒート：** FAST状態（SPEED 70以上）になると被ダメ軽減が消え、通常攻撃が-10%になるが、ガード不可になる。「遅くて硬い」か「速くて軽いがガード貫通」のトレードオフ。

---

**LYRA-X 🧬 ／ バイオ・エボリューション**
- ターン終了時、固定**2HP**を自動回復する（v6弱体化：旧3HP）
- 通常攻撃ダメージ-10%（v6弱体化追加）
- スロー・回復封じの効果は**1ターン持続**（v6弱体化：旧2ターン。他キャラと同じ）
- **腐食：LYRA-Xが対戦相手である間、相手がSLOW状態（SPEED 35以下）であればターン終了時に固定**3ダメージ**を与える（v6弱体化：旧5ダメージ）。SLOW状態になった原因は問わない。**（確定仕様）
- 細胞再生：与えたダメージの30%をHP吸収
- コンセプト：デバフを軸に長期戦を制するキャラ。スロー持続が1Tになったため、継続的なスロー維持にはゲージ管理が必要になった。

---

**BLAZE 🔥 ／ エクスプローシブ（爆弾魔）**
全攻撃ダメージ+30%（dmgMod:1.3、v6.1変更）。**属性付き攻撃には追加で専用×1.4倍率**が乗る（v6.1追加）。**属性付き攻撃・固有技使用時は確定で自爆**が発生する。

| 自爆の仕様 | 内容 |
|------------|------|
| 自爆発生条件 | 属性付き攻撃・固有技使用時（確定） |
| 自爆ダメージ | 与えたダメージの**20%**を自分も受ける（v6.1変更：自爆リスク軽減で攻撃的に使いやすく） |
| 誘爆 | 自爆発生時、自爆ダメージの50%を相手にも与える（道連れ効果） |
| 自爆のボーナス | 発生ターンのゲージ増加量が2倍 |
| 自爆の回避 | 起爆準備を使った次のターンは自爆なし・確定クリティカル（速度特攻・ガード破壊・回復封じ・爆裂拳すべてに適用。v6.8バグ修正済み） |

**BLAZE 🔥 ／ 起爆準備（ガードの置き換え）**
- そのターンは攻撃しない代わりに受けるダメージを30%軽減
- 次のターンの確定クリティカル（ダメージ1.3倍）＆自爆なし
- **起爆準備の効果は「攻撃系行動（種類問わず）」をした瞬間に消費される。加速・スロー・回復・挑発などの非攻撃行動では持続する。**
- カウンターに弱い：攻撃+50%のため、相手にカウンターを構えられると大ダメージが返ってくる。起爆準備で様子見するのがカウンター対策。

## 4. 速度システム（SPEED）

### 4-1. 基本仕様

| 条件 | 効果 |
|------|------|
| SPEED差10以上 | 速い方が先に行動解決。先手で後手HP0になると後手の行動キャンセル |
| SPEED差10未満 | 同時解決（旧ルール） |

| SPEED 35以下（SLOW） | 自分の攻撃ダメージが-20%。TITANは被ダメージ-15%・与ダメ+10%（v6.9：旧被ダメ-25%・与ダメ+20%） |

| 毎ターン終了時 | SPEED50以上の場合のみ、50との差の15%分だけ50に向かって戻る。SPEED50以下は維持。 |

### 4-2. 速度特攻のボーナスダメージ

| 相手のSPEED | 追加ダメージ |
|-------------|-------------|
| 70〜74（FAST） | +8 |
| 75〜84 | +14 |
| 85〜100 | +20 |

### 4-3. オーバークロック
- コスト：ゲージ100%消費
- 効果：次のターンのみ2回行動が確定
- リスク：使用後3ターンはゲージ増加量が50%に減少

## 11. ネット対戦仕様（ルームマッチ特化）

- **ランダムマッチは初期リリースでは見送り**
- **合言葉（数字4桁など）によるプライベートマッチ**に特化
- Firebase Realtime Databaseを使用
- **同期する情報：「両者が確定ボタンを押したフラグ」＋「命中判定結果フラグ」**
- 演出・ダメージ計算は各端末で独立処理（通信量を最小化）

### 11-1. 乱数同期ルール（v5.1新規）

命中判定など乱数を用いる処理を各端末で独立して実行すると、P1側「ヒット」・P2側「ミス」という結果のズレが生じゲームが進行不能になる。これを防ぐため以下のルールを採用する。

| 役割 | 処理内容 |
|------|---------|
| **ホスト（ルーム作成側 / Player 1）** | 命中判定・乱数計算を実行し、「結果フラグ」を通信パケットに含めて送信 |
| **ゲスト（Player 2）** | ホストから受信した結果フラグに従ってダメージ処理・アニメーションを再生（自端末では乱数計算しない） |

**対象となる乱数処理：**
- 中・強攻撃の命中判定（80% / 70%）
- カウンターの返し命中判定（80%）
- 速度特攻のSPEED低下量（10〜15）
- 加速のSPEED上昇量（20〜30）
- 回復のHP回復量（15〜25）

## 12. CPU AI仕様

### 12-1. 難易度定義

| 難易度 | 挙動 |
|--------|------|
| EASY | 行動ウェイトなし、ほぼランダム。システム学習用 |
| NORMAL | キャラごとの行動ウェイトに従って動く標準モード |
| HARD | プレイヤーのHP・ゲージ・バフ状態を参照し、最適なメタ行動を優先選択 |

### 12-2. キャラクター固有の行動ウェイト（NORMAL / HARD共通：abSmartAct）

HARDはNORMALと同じabSmartAct関数を使用。キャラごとの戦略的ウェイトで動く。

**NEXUS-7 🤖**
- ゲージ25%以上で属性攻撃3種を積極選択（各+8）
- 学習済みフラグがある場合、対応する属性攻撃を最優先（+10追加）
- 量子強化中はさらに属性攻撃を加重（+6追加）
- ゲージ50%以上で量子ビームも選択（+6）
- 戦略：学習→同属性攻撃×2.0の連鎖を軸に属性攻撃を多用

**SHADOW 🥷**
- SPEED<70かつゲージ15%以上で加速を最優先（+8）
- FAST未達成ならゲージ15%以上で加速（+5）
- 強攻撃を常に高ウェイト選択（+6）：hitMod+10で80%命中を活かす
- FAST状態でゲージあればカウンター（+3）：控えめ
- ゲージ25%以上でガード破壊・回復封じ（各+5）
- NOVA QUEEN相手はさらにガード破壊・回復封じを追加（各+5）
- 戦略：加速でFASTを維持しつつ強攻撃80%ヒットで削る。NOVA対策に属性攻撃を重ねる

**NOVA QUEEN 🔮**
- **ゲージ70%未満フェーズ：** mid/strong/属性攻撃/accel/guard/slow/tauntを選択しない。弱攻撃・カウンター・回復（HP低下時）・星屑砲（ゲージ50%以上）のみ
  - 相手が前ターンに攻撃してきた場合はカウンターを積極選択（+6〜10）
  - HP低下に応じて回復を優先（HP40%未満+10、60%未満+6、75%未満+3）
- **ゲージ70%以上フェーズ：** mid/strong/accel/guard/slow/tauntを除外。属性攻撃3種（各+10）・星屑砲（+8〜14）・カウンター・回復を中心に動く
- 戦略：ゲージ70%到達まで弱攻撃とカウンターで耐え、70%到達後はコスト10%で属性攻撃を連発。回復とカウンターでHPを管理しながら削る

**TITAN 🦾**
- ゲージ50%以上で超電磁砲（+4）
- 瀕死（HP40以下）かつゲージ50%以上で超電磁砲（さらに+4）
- ゲージがあればカウンター（+1）、中攻撃（+2）：ゲージを通常攻撃で稼ぐ
- ゲージ25%以上で属性攻撃3種（各+2）
- 「加速」は基本封印：LYRA-X相手で自分がSLOW中のみ使用（+8）
- 「強攻撃」は相手SPEED50未満のときのみ選択（速度差で外れるため）
- LYRA-X相手でSLOWの場合は加速最優先（腐食回避のため）
- 戦略：通常攻撃でゲージを貯め、超電磁砲と属性攻撃で削る重戦車スタイル（v6.9：超電磁砲ダメ調整・SLOW時バフ縮小で他キャラと対等な勝率に）

**LYRA-X 🧬**
- ゲージ30%以上でスロー最優先（相手未SLOW時+12、SLOW中は+4）
- ゲージ25%以上で回復封じ（+10）：長期戦制圧の軸
- 相手がSLOW中かつゲージ25%以上でガード破壊・速度特攻も選択（各+4）
- ゲージ50%以上で細胞再生（+8）
- 自分がSLOW中かつゲージ15%以上で加速（+6）
- 戦略：スロー→腐食ダメージ＋回復封じで長期戦を制圧。細胞再生で体力差を広げる

**BLAZE 🔥**
- 起爆準備済み（primed）かつゲージ50%以上で爆裂拳を強く優先（+14）
- 起爆準備済みかつゲージ25%以上で属性攻撃3種も選択（各+6）
- 未起爆でゲージ30%以上は起爆準備（guard）を高ウェイトで選択（+9）
- 未起爆でゲージ30%以上かつ封じなしは挑発も選択（+4）
- 戦略：挑発→起爆準備→爆裂拳or属性攻撃のギャンブルコンボが主軸

### 12-3. グローバルAIルール（全キャラ共通の上乗せ）

キャラ固有ウェイトに加えて、以下の状況判断が全キャラに適用される：

| 状況 | 追加ウェイト |
|------|-------------|
| 相手がTITAN＆ゲージあり | カウンター+5（超電磁砲対策） |
| 相手がNOVA＆ゲージあり | 回復封じ+6（回復・星屑砲をヘイトして封じる） |
| 相手がSHADOW＆FAST状態＆ゲージあり | 速度特攻+8（FAST時のボーナス狙い） |
| 相手がSHADOW＆SLOW状態＆ゲージあり | ガード破壊+8・回復封じ+8（FAST消失の隙を突く） |
| 相手が起爆準備（BLAZE primed）状態 | ゲージ50%以上→カウンター+3。不足→ガード+3・スロー+2 |
| 相手ゲージ50%以上 | ガード+2・カウンター+2（**自分ゲージ不足時はカウンター除外**）（v6.9バグ修正） |
| 相手が前ターンにカウンター使用 | mid+4・strong+4・属性攻撃+4（無防備ターンを攻める） |
| 相手の直近5ターンにカウンター2回以上 | mid+3・strong+3（カウンター癖への対処） |
| 相手の直近5ターンにガード2回以上 | ガード破壊+5 |
| 相手の直近5ターンに加速2回以上 | 速度特攻+5 |
| 相手の直近5ターンに回復2回以上 | 回復封じ+5 |
| 相手ゲージ75%以上 | ガード+2・カウンター+2（固有技警戒） |
| 自分がSLOW状態＆ゲージ15%以上 | 加速+6（汎用） |

### 12-4. HARD AIのコンセプト（v6.8）

HARDはabSmartActに完全委任。NORMALと同じウェイトで動くが、キャラの持ち味を最大限発揮する戦略的な重み付けにより、プレイヤーが単純な対策を取りにくい動きをする。

**ファジー行動のコンセプト：**
重み付き抽選（ランダム選択）なので100%確定的な行動にはならない。「起爆準備したら必ずカウンター」という完全な読み合い対策はできず、ある程度の読み合い要素が残る。

- カウンター連続ペナルティの50%コストが適切か
- タイマー30秒が短すぎ/長すぎないか
- ネット対戦：各端末独立処理で乱数がズレた場合の対処（シード共有の必要性）

## 15. 開発者モード（非公開機能）

### 15-1. 起動条件（いずれか）

| 方法 | 詳細 |
|------|------|
| URLパラメータ | `?dev=1` または `?mode=dev` をURLに付加 |
| ローカルストレージ | ブラウザコンソールで `localStorage.setItem('dev_mode','1')` を実行 |
| 隠しキー入力 | タイトル画面でキーボードから `SYNTAX` とタイプ（トグル） |

**推奨手順（開発時）：**
1. ブラウザで `syntax_wars_v62.html?dev=1` を開く（最速）
2. または一度コンソールで `localStorage.setItem('dev_mode','1')` を実行 → 以降はリロードのみでON
3. OFFにしたい場合はタイトル画面で `SYNTAX` とタイプ、またはコンソールで `localStorage.removeItem('dev_mode')`

起動中はタイトルロゴの横に `[DEV]` バッジを表示。`SYNTAX` 再入力でトグルOFF（localStorage も同時更新）。
